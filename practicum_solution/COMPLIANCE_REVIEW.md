# Замечания по соответствию ТЗ

Все ранее выявленные несоответствия закрыты:

1. Генерация/бенчмарки теперь стримовые, по умолчанию создают 10+ млн лайков, а отчёт `reports/performance.md` фиксирует результаты прогона с этим объёмом.
2. Скрипты приведены к требованиям wemake-python-styleguide и mypy (убраны лямбды, длинные строки, добавлены аннотации и поясняющие докстринги).
3. Мониторинг ошибок реализуется через официальный облачный Sentry: `.env.example` содержит пустой `SENTRY_DSN`, в README расписано, как создать проект и вставить его DSN, чтобы SDK сразу отправлял события в Sentry без самодельных приёмников.
4. Filebeat/Logstash адаптированы под формат uvicorn: docker-json декодируется, grok разбирает строки и сохраняет разметку в Elasticsearch.
5. `docker compose up -d` автоматически наполняет MongoDB/PostgreSQL данными и запускает бенчмарк, поэтому API и метрики можно проверить одной командой.
